# Difference-Objects-Classes
In Java, it's important to understand what classes and objects are. These two work together, but are not the same thing. 
A class to me is like a blueprint. It is a template you write in your code that defines what something will look like and what it can do. It does not actually exist as anything real yet. It is just the plan. Think of it like the design documents for a subamrine. The blueprints describes everything of what the boat is going to be like or have. But you can't go out to sea with just a blueprint alone. 
This is where object comes in. An object is what actually gets built from that blueprint. Just like a shipyard takes the design documents and constructs an actual submarine from them. Java takes your class and uses it to create something real in memory when your program runs. That something is the object. It has actual values stored in it and takes up space in memory. The blueprint just decribes what it will be, and the object is the real thing that exists and works. 
You can create as many objects as you want from one class, and each one hold its own data independently. So if Subamrine is the class, then USS Newport News and USS Alexandria are two separate objects with their own names, crew sizes, and statuses. Changing one does not affect the other at all. 
The main differences between classes and objects is that a class exisit in your code, while an object exists in memory when the program runs. You write a class once, and from that class you can write as many objects from it as you need, whenever you need them. 
In my code I created one Submarine class with the characteristics like name, crewSize, and status. After I created the Subamrine class, I created three separate submarine objects from it. Each object hold its own data, but they all share the same structure defined by the class.
